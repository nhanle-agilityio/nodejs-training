# 🧪 Discovering Node.js Test Runner

## 🇬🇧 English Version

### 1. What is a Test Runner?

A test runner is a tool that executes your tests and reports whether they pass or fail.

- **Built-in:** Node.js (since v18) includes a native test runner. You don't need external libraries like Jest or Mocha for basic testing.

### 2. Why Test?

- **Verification:** Ensures code works as expected.
- **Bug Catching:** Finds issues early.
- **Reliability:** Makes code maintainable and robust.

### 3. Basic Usage

You can run tests using the `node --test` command. It automatically looks for files matching patterns like `test/**/*.js`, `**/*.test.js`.

## 🇻🇳 Vietnamese Version

### 1. Trình chạy thử nghiệm (Test Runner) là gì?

Test Runner là công cụ giúp thực thi các bài kiểm tra (test) và báo cáo kết quả (đạt/trượt).

- **Tích hợp sẵn:** Từ Node.js v18 trở đi, đã có sẵn test runner nội bộ. Bạn không cần cài thêm thư viện ngoài như Jest hay Mocha cho các nhu cầu cơ bản.

### 2. Tại sao phải test?

- **Xác minh:** Đảm bảo code chạy đúng ý đồ.
- **Bắt lỗi:** Phát hiện lỗi sớm trong quá trình phát triển.
- **Độ tin cậy:** Giúp code dễ bảo trì và bền vững hơn.

### 3. Cách dùng cơ bản

Bạn có thể chạy test bằng lệnh `node --test`. Node.js sẽ tự động tìm các file theo mẫu như `test/**/*.js`, `**/*.test.js`.
