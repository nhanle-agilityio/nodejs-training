# demo-auth

Generated by AI - A minimal frontend (Vite + React + TypeScript + `@clerk/clerk-react`) to **sign in with Clerk** and **retrieve JWTs** for testing the Nest API in `../nest-example`.

## 1. Setup

1. Open Clerk Dashboard and select the **same** app used for `CLERK_SECRET_KEY` in `nest-example/.env`.
2. Copy the **Publishable key** (`pk_test_...`).
3. Update `demo-auth/.env.local`:

```env
VITE_CLERK_PUBLISHABLE_KEY=pk_test_your_key
VITE_API_BASE_URL=http://localhost:3000/api/v1
```

> The publishable key and Nest `CLERK_SECRET_KEY` **must belong to the same Clerk app**, otherwise the token will return `401` in Nest.

## 2. Run

```bash
cd demo-auth
npm install
npm run dev
```

Open `http://localhost:5173`.

## 3. Get JWT for API Testing

1. Click **Sign in** and log in with a test user (for role-based testing, the user should have `publicMetadata.role = "admin"`).
2. In the post-login panel, click **Get / Refresh JWT** and then **Copy token**.
3. Paste it in Postman / Thunder Client: **Authorization -> Bearer Token**.

## 4. Call Nest API Directly from the UI

The panel includes 3 test buttons using the header `Authorization: Bearer <jwt>`:

- `GET /products` - public (via `@Public()`)
- `POST /products` - requires a valid Bearer token
- `DELETE /products/:id` - requires Bearer token + `admin` role (ID entered by user in the UI)

## 5. Expected Result Matrix

| Request | Expected |
| --- | --- |
| GET `/products` without Bearer | 200 |
| POST `/products` without Authorization | 401 |
| POST with valid Bearer | 2xx |
| DELETE with non-admin token | 403 |
| DELETE with admin token | 2xx |
